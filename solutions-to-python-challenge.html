Problem 1: 274877906944

由图中的映射关系，K->M，O->P，E->G，可以看出它是将字母表中的每个字母顺次往后平移两次。对下方的一串文字进行此操作即可。 

import string

hints = '''g fmnc wms bgblr rpylqjyrc gr zw fylb. rfyrq ufyr amknsrcpq 
ypc dmp. bmgle gr gl zw fylb gq glcddgagclr ylb rfyr'q ufw rfgq rcvr 
gq qm jmle. sqgle qrpgle.kyicrpylq() gq pcamkkclbcb. lmu ynnjw ml rfc spj.'''
source = string.ascii_lowercase
table = str.maketrans(source, source[2:] + source[:2])
print(hints.translate(table))
# i hope you didnt translate it by hand. thats what computers are for. 
# doing it in by hand is inefficient and that's why this text is so long. 
# using string.maketrans() is recommended. now apply on the url.
print("map".translate(table)) # ocr
# http://www.pythonchallenge.com/pc/def/ocr.html

在看到提示 “using string.maketrans() is recommended” 之前，我是在手动构造一个字典映射关系，但是就遇到了问题，因为字符串 str 是不可变对象， 每次对一个字符进行替换之后就要去考虑应该如何去保存，后来就想到每次都调用 str.replace() ，这样手动去实现一个映射替换太麻烦了。

str.maketrans() 这是一个 python 中的内建函数（ built-in function ），在标准库中的描述如下：

maketrans(x, y=None, z=None, /) Return a translation table usable for str.translate().

If there is only one argument, it must be a dictionary mapping Unicode ordinals (integers) or characters to Unicode ordinals, strings or None. Character keys will be then converted to ordinals. If there are two arguments, they must be strings of equal length, and in the resulting dictionary, each character in x will be mapped to the character at the same position in y. If there is a third argument, it must be a string, whose characters will be mapped to None in the result.

str.maketrans() 就是去构造一个映射表( table )，这个映射表定义了 str.translate() 转换关系。根据 maketrans() 参数的个数，可以分为以下三类：

1.仅有一个参数时，就必须是一个字典，将 Unicode 字符序或者字符映射到 Unicode 字符序。 

print('!1@2#3$4'.translate({ord(i):None for i in '!@#$'}))  # 删除：1234

2.有两个参数时，这两个参数为等长度的字符串，在进行映射时就是将前一个字符串某一位置上的字符变换为后面字符串相同位置上的字符。 

import string

table = str.maketrans(string.ascii_lowercase, string.ascii_uppercase)
print('https://aprilwith.me'.translate(table))  # 'HTTPS://APRILWITH.ME'

3.有三个参数，如果存在第三个参数，它定义了一些应该被删除（或者理解为过滤）掉的字符。 

import string

table = str.maketrans(string.ascii_lowercase, string.ascii_uppercase, '!@#$%^&*')
print('h@%tt^##ps:%*@/%^/&ap@ri*l(#wi^&t*@(&h^*%^&%.^me&*'.translate(table))  # 'HTTPS://APRILWITH.ME'

str.translate(table) 也是内建函数，描述如下：

S.translate(table) -> str Return a copy of the string S, where all characters have been mapped

through the given translation table, which must be a mapping of Unicode ordinals to Unicode ordinals, strings, or None. Unmapped characters are left untouched. Characters mapped to None are deleted.

str.translate(table) 就是使用 table 所定义的映射关系，在所调用的字符串上执行映射，可能是 Unicode 字符互相映射， 也可能是映射为 None ，即删除这个字符。那些没有定义映射关系的字符，保持。

注1：还需要一些说明的是，在 Python3 中， string.maketrans() 已经被 deprecated 了，转而使用静态方法来替代 static str.maketrans() 。现在，对 str ， bytes ， bytearray 每个都有着与之所对应的 maketrans() 和 translate() 方法， 这样就解决了原先 string 模块中 string.maketrans() 函数到底支持的类型的问题。

注2：要区分 string 类型和 bytes 类型。在 Python3 中， string 不是 bytes 。 string 是 Unicode ，使用 "..." 或者 '...' 表示；而 bytes 是 bytes ，使用 b"..." 或者 b'...' 表示。

注1和注2均参考 How come string.maketrans does not work in Python 3.1?

Problem 2: ocr

这道题目自己之前一直不敢去做，原因是一直琢磨着去用正则表达式去解决，但有时候一想到正则就会犯怯，这需要去克服；后来想想，这个都是 pythonchallenge ，当然只要是使用 Python 来解决这个问题就行了，优雅不优雅暂先不提及，能够求解出答案就可以，代码的效率与是否 Pythonic 都可以之后再去讨论，解决问题才是王道。

recognize the characters. maybe they are in the book, but MAYBE they are in the page source.

根据提示，查看页面源码，下方包含很大段的注释 find rare characters in the mess below: ... ，需要在注释中的一大堆字符中找到出现次数最少的字符。 

import string
import requests

r = requests.get("http://www.pythonchallenge.com/pc/def/ocr.html")
t = r.text
comment_begin = t.find("%%$@")

result = {}
for char in t[comment_begin:]:
    result[char] = result.get(char, 0) + 1

print(result)

结果返回为个字符对应的出现次数

{'@': 6157, '-': 2, '{': 6046, 't': 1, 'u': 1, 'e': 1, '_': 6112, ']': 6152, ')': 6186, '$': 6046, '[': 6108, '*': 6034, '%': 6104, '}': 6105, '(': 6154, '>': 1, 'y': 1, 'a': 1, '#': 6115, '!': 6079, 'l': 1, 'q': 1, '&': 6043, '\n': 1222, 'i': 1, '+': 6066, '^': 6030}

可以看出，对应有八个字符仅出现过 1 次，为提示中的 rare characters ：

't' 'u' 'e' 'y' 'a' 'l' 'q' 'i'

但这八个字符可以有很多种组合，最后决定以这些字符在这一大段注释中出现的先后顺序组织： 

rare_string = "tueyalqi"
url = []
for char in t[comment_begin:]:
    if char in rare_string:
        url.append(char)
url = ("").join(url)
print("http://www.pythonchallenge.com/pc/def/{0}.html".format(url))
# http://www.pythonchallenge.com/pc/def/equality.html
