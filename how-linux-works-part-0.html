首先是 Shell

对于 Shell，这里我们引用池建强在《终极 Shell》里的通俗介绍：

Shell是Linux/Unix的一个外壳，你理解成衣服也行。它负责外界与Linux内核的交互，接收用户或其他应用程序的命令，然后把这些命令转化成内核能理解的语言，传给内核，内核是真正干活的，干完之后再把结果返回用户或应用程序。

标准输入输出

以下内容摘自《精通 Linux，第二版》

使用 cat 命令来学习 Unix 的输入和输出（以下简称 I/O）。Unix 进程使用 I/O 流来读写数据。进程从输入流中读取数据，向输出流写出数据。数据流非常灵活，比如输入流可以是文件、设备、终端，甚至还可以是来自其他进程的输出流。

想知道输入流的工作原理，只需要输入 cat 命令并回车，这时候就会看到屏幕上没有显示任何结果，因此 cat 命令仍在运行中。现在输入几个字符然后回车，会看到 cat 命令在屏幕上显示出刚刚输入的字符。最后，可以在任意空白行按 CTRL-D 终止 cat 命令的执行并回到 shell 提示符。

刚刚和 cat 命令进行的一系列交互就是通过数据流机制来实现的。因为没有指定输入文件名，cat 命令就从 Linux 内核提供的默认 标准输入流 中获得输入数据，这时运行 cat 运行的终端就成为标准输入。

注解：按 CTRL-D 终止当前终端的标准输入并终止命令（通常会终止一个程序）。这和 CTRL-C 不一样。 CTRL-C 是终止当前进程的运行，无论是否有输入和输出。

标准输出 也与之类似。内核为每个进程提供一个标准输出流供它们输出数据。cat 命令在终端运行的时候，标准输出就该终端建立连接，cat 命令将数据输出到标准输出，就是在屏幕上看到的结果。

标准输入和标准输出通常简写为 stdin 和 stdout 。很多命令和 cat 一样，如果不为它们指定输入文件，就从标准输入获得数据。输出则有点不同，一部分命令将数据输出到标准输出，另一部分命令将数据直接输出到文件。 除了标准输入和输出外，还有标准错误信息流。

标准流的一个优点是可以随心所欲地指定数据的输入输出来源。

常用的基础命令

注：这些基础命令大都需要输入参数，同时支持可选项和格式，这里仅将常用的列出。

  • man
    • man program will show you the on-line documentation (the man page) for a program (e.g., try man man or man ls). Handy if you want to know what a program does.
    • man function to see documentation for standard library functions. The command man -k string will search for man pages whose titles contain string.
    • Sometimes there is more than one man page with the same name. In this case man -k will distingiush them by different manual section numbers, e.g., printf (1) (a shell command) vs. printf (3) (a library routine). To get a man page from a specific section, use man section name, e.g. man 3 printf.
  • ls
    显示指定目录的内容，默认参数为当前目录
    • 指定目录 ls /some/other/dir
    • 以长列表形式输出显示详细信息 ls -l
    • 显示文件类型信息 ls -F
  • cp
    复制文件 cp old-name new-name （make a copy of a file）
  • mv
    • 重命名文件 mv file1 file2 （change the name of a file）
    • 移动文件到目录 mv file1 ... fileN dir （move files between directories）
  • touch
    用来创建文件，如果文件已经存在，则会更新文件的时间戳信息
  • rm
    • 删除文件 rm file deletes a file. Deleted files cannot be recovered. Use this command carefully
    • 为了提醒我们，在 .zshrc 中配置 alias rm="rm -i" ，再次输入 rm file-name 时就会出现提示 rm: remove regular file 'file-name'? ，这是我们就能有几秒的思考时间来确实是否真的要删除掉这个文件。
  • echo
    将其参数显示到标准输出（一般与上面的 Shell 通配符 结合使用）
  • cd
    • 设置当前工作目录（当前工作目录是指进程和 Shell 当前所在的目录） cd dir
    • 不带 dir 参数，默认进入个人的主目录
  • mkdir
    创建新目录
  • rmdir
    • 删除空目录（如果目录里面有内容，这个命令会执行失败）
    • rm -rf not-empty-dir 删除一个目录以及其中所有内容。 Cautious ： -r 选项会依次递归删除 dir 中的所有文件和子目录， -f 选项表示强制删除。使用 -rf 时不要在参数里使用通配符，并且执行命令前最好检查参数是否正确
  • pwd
    输出当前的工作目录名，有以下两个用处：
    • 并不是所有的提示符都显示当前目录名，甚至有时候需要摆脱它，因为其占用很大空间，这时候就需要使用 pwd 来解决
    • 使用符号链接的时候通常很难获知当前目录信息，可以使用 pwd -P 来查看
  • chmod
    更改一个文件或目录的权限信息。以下是一些简单例子：
    • chmod 644 file; owner can read or write the file, others can only read it
    • chmod 600 file; owner can read or write the file, others can’t do anything with it
    • chmod 755 file; owner can read, write, or execute the file, others can read or execute it. This is typically used for programs or for directories (where the execute bit has the special meaning of letting somebody find files in the directory)
    • chmod 700 file; owner can read, write, or execute the file, others can’t do anything with it
